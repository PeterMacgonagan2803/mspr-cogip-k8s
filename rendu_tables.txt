--- TABLE 14 ---
Difficulté | Impact | Solution
Provider Terraform telmate/proxmox : erreurs de permissions VM.Monitor même avec tokens privilégiés | Blocage complet, Terraform ne pouvait pas créer les VMs | Migration vers le provider bpg/proxmox qui gère mieux l'authentification par mot de passe et le SSH natif
Packer + ISO : l'installeur manuel se lançait au lieu du mode cloud-init autoinstall | Impossible de créer le template automatiquement | Abandon de Packer + ISO au profit d'un script utilisant l'image cloud Ubuntu qcow2 avec cloud-init natif
Images Docker bitnami/* : tags retirés ou introuvables pour Odoo et PostgreSQL | CrashLoopBackOff, pods impossibles à démarrer | Utilisation des mêmes tags sur bitnamilegacy/* + global.security.allowInsecureImages: true dans les valeurs Helm
Helm : Service LoadBalancer du chart Odoo en <pending> sans cloud-controller K3s | Odoo non accessible via IP externe | Surcharge service.type: ClusterIP ; accès utilisateur uniquement via Ingress Traefik
Timeout SSH lors d'Ansible depuis Windows vers les VMs via Proxmox | Playbooks échouent ou se coupent en cours | Exécution d'Ansible directement sur le serveur Proxmox avec scripts Python/Paramiko pour l'orchestration
Traefik / HTTP : l'Ingress ne routait que websecure (443), odoo.local inaccessible en HTTP | Accès impossible depuis certains navigateurs | Annotation traefik.ingress.kubernetes.io/router.entrypoints: web,websecure sur l'Ingress Odoo
QEMU Guest Agent : Terraform bloquait indéfiniment en attendant l'agent non installé | terraform apply ne terminait jamais | Désactivation dans la configuration Terraform (agent { enabled = false })
Réseau Proxmox : VMs sur réseau privé vmbr1 non accessibles depuis l'extérieur | Odoo inaccessible depuis le navigateur client | vmbr1 + MASQUERADE pour la sortie ; DNAT PREROUTING sur vmbr0 (ports 80 et 443 → 10.10.10.10) ; persistance via iptables-persistent
